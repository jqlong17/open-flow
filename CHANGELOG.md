# 更新日志（简体中文）

本文件用于快速记录 Open Flow 近期的重要变更（按日期倒序）。

## 2026-04-06

- 新增面向 **Mac App Store** 的 macOS 版本，开始形成「GitHub 开源完整版 + 商店精简版」的双版本发布形态。
- 商店版当前聚焦 **本地离线麦克风转写** 这条主路径，收敛了会议模式、草稿整理等不适合当前商店版本的功能入口，降低安装理解成本与审核风险。
- 商店版改为**内置离线模型**，安装后即可直接使用，不再依赖首次启动再下载必需模型。
- 优化 macOS 设置页与日志页的信息展示，移除商店版不需要暴露的会议/屏幕录制相关入口，使界面更符合当前 MVP 范围。
- 优化 macOS 权限授权体验：当辅助功能、输入监控、麦克风权限完成授权后，应用会自动尝试恢复 daemon 并刷新状态，减少手动重启步骤。
- README 补充双版本说明，明确商店版付费的原因主要是 Apple 开发者会员、签名、分发与维护成本；同时保留 GitHub 开源和自行构建路径。

## 2026-03-29

- 修复 macOS 麦克风授权链路不稳定的问题：设置页现在会由前台窗口主动请求麦克风权限，不再依赖后台 daemon 触发系统弹窗。
- 将权限检查结果从简单布尔值升级为结构化状态，现可区分 **未请求 / 已拒绝 / 受限制 / 已授权 / 未知**，降低权限误判带来的排障成本。
- 调整 daemon 的权限策略：后台进程不再尝试主动请求麦克风权限，而是明确引导用户在 **Permissions** 页完成前台授权。
- 优化麦克风低振幅提示，区分“权限未通过”和“权限已通过但当前输入设备几乎无声音”两类问题，减少误导性报错。
- 降低设置页权限轮询强度，避免频繁拉起短命子进程导致 TCC 状态抖动和调试噪音。

## 2026-03-26

- 重整 macOS 设置页的信息架构，重新划分为 **General / Recognition / Models / Vocabulary / Meetings / Permissions / Logs** 七个分页，让识别、模型、个人热词、会议与日志职责更清晰。
- 清理旧版 `Diagnostics` 遗留实现，将会议概览与会话状态收敛到 `Meetings`，将排障、热键、daemon 与性能相关内容收敛到 `Logs`，减少重复卡片和历史过渡代码。
- 修复 macOS 发布打包脚本错误优先选择 `Apple Development` 证书的问题，正式发版路径现已切换为优先使用 `Developer ID Application`。
- macOS Release 打包流程改为使用 `ditto` 生成 `.app.zip`，并增加 notarization（公证）检查与可选提交流程，降低下载后“无法安装/无法打开”的风险。
- 新增 `scripts/setup-notary-profile.sh`，用于本机初始化 `notarytool` profile，方便后续正式发布时直接走签名 + 公证链路。
- 新增 **桌面音频 + 麦克风（会议）** 录音模式，可同时采集系统声音与麦克风输入，并输出带“我 / 对方”标签的首版会议转写结果。
- 系统音频 helper 增加可捕获 display 的重试与回退策略，并在 helper 启动后加入早期失败检测，提升会议模式下系统音频链路的稳定性。
- 双路会议模式已支持**持续分段采集、持续分段转写、持续落盘**，会按固定时长生成片段并持续写入本地会话目录。
- 新增本机会话落盘结构，会议模式会持续写入 `session.json`、`transcripts.jsonl`、`merged_transcript.md` 以及 `segments/*.wav`，便于长会议复盘和后续再处理。
- 调整设置页录音相关交互，将“实验捕获模式”正式收敛为上方的 **录音模式** 入口，并将“输入源”明确为 **麦克风输入源**，减少配置理解成本。
- 更新 README 中的 macOS 设置页截图与说明，补充会议模式、持续分段与持续落盘的使用说明。

## 2026-03-25

- 录音草稿新增 **AI 转换**能力，支持在草稿面板右侧展开内嵌 AI 区域，不再弹出独立窗口。
- 新增三个默认转换能力：**去除口语**、**转为字幕**、**转为 MD**，并支持在同一面板中直接生成、复制、清空结果。
- AI 转换提示词支持本地保存与重置，便于用户按自己的写作、字幕或笔记习惯持续调整。
- 转字幕能力默认按文本量自动估算时长，并以标准 `SRT` 格式为目标，后续还会继续扩展更多 AI 转换模板。
- 抽离通用 LLM 调用层，统一复用智谱模型配置，为后续在纠错之外扩展更多 AI 能力打基础。
- 新增独立的性能日志能力，可按 session 持久化记录录音、ASR、LLM 修正、输出与端到端耗时，以及 CPU / RSS / VSZ 等进程资源快照，便于后续持续做性能优化与复盘。
- 性能日志单独落盘到本机 `performance/*.jsonl`，并引入基础 schema version 与日志保留策略，避免与普通 `daemon.log` 混杂。
- 设置页补充性能日志开关与开发期入口，但将这部分 UI 改为**开发者专用编译期开关**控制，默认不会出现在正式发布构建中。
- 新增 `OPENFLOW_PERF_DEV_UI=1` 本地构建方式，方便协作者在开发环境中启用性能调试模块，同时保持 release 包面向最终用户的界面简洁。
- README 补充开发者说明，明确性能日志模块的用途、本地开启方式、日志目录以及正式发版默认关闭的策略。

## 2026-03-24

- 修复 macOS 发布打包时设置助手的二进制路径探测逻辑，避免 release 包内仍携带旧版设置界面。
- 更新 macOS 配置页面视觉样式，采用更精致的侧边栏与更紧凑的卡片布局，统一了设置面板的整体观感。
- 新增 `Vocabulary` 独立分页，将个人热词、纠错开关、纠错模型与智谱 API Key 配置集中到同一页。
- 增加个人热词能力，支持按行维护术语、人名、产品名和项目代号，降低 ASR 对专有词的误识别。
- 增加基于大模型的输出修正流程，可在 `偏好设置 -> Vocabulary` 中开启 **Enable correction**，对转写结果做轻量后处理。
- 在 `Vocabulary -> API key` 中新增智谱 BigModel 申请入口，方便用户申请并配置 API Key 后体验热词纠错。
- 模型页新增直接打开模型文件夹的能力，便于检查本地模型目录和分享路径。
