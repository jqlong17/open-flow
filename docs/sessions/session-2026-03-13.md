## 关键决策

| 决策 | 选项 | 原因 |
|------|------|------|
| FFT size | 512（非 400） | 与 Kaldi next-power-of-2 对齐 |
| MEL_FMAX | 8000Hz | Nyquist for 16kHz 音频 |
| Dither | 默认关闭 | 匹配 FunASR dither=0.0 |
| Log floor | `x.max(f32::EPSILON).ln()` | 匹配 torchaudio 行为（约 -15.94） |
| use_best_non_blank | 默认 false | 避免特征不精确时产生垃圾输出 |
| daemon 实现 | setsid + PID 文件 | 避免 launchd 复杂性，适合开发阶段 |
| 剪贴板恢复 | 不恢复旧内容 | 用户明确要求，transcribed text 保留在剪贴板 |
| 记忆系统 | Cursor hooks + 手动触发 | sessionStart 自动注入 CURRENT_STATE.md；每日首次对话提示更新；用户输入“更新索引”时执行更新 |

## Cursor Hooks 记忆系统（本次对话第二段）

### 背景
用户在 Cursor 设置界面截图中看到「Hooks」功能，询问是否可用于自动更新知识索引。

### 实现内容

1. **`.cursor/hooks.json`**：配置两个 hook
   - `sessionStart`：对话开始时自动注入 `CURRENT_STATE.md`
   - `stop`：对话结束时触发 AI 更新记忆（`loop_limit=1`）

2. **`.cursor/hooks/session-start.sh`**
   - 读取 stdin JSON，提取 `workspace_roots[0]`
   - 若无则 fallback 到相对路径 `docs/CURRENT_STATE.md`
   - 用 `jq -Rs` 将文件内容安全序列化为 JSON 字符串后输出

3. **`.cursor/hooks/session-stop.sh`**
   - 检查 `loop_count` 避免无限触发
   - 仅 `status=completed` 时发出 follow-up

4. **`.cursor/rules/memory-system.mdc`**：元规则，告诉 AI 何时/如何更新记忆

### 验证结果
- `session-start.sh` 输出合法 JSON，`additional_context` 包含完整 `CURRENT_STATE.md` 内容 ✅
- `session-stop.sh` 输出 `followup_message` 字段，`loop_count!=0` 时返回 `{}` ✅
